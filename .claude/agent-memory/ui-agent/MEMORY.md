# ui-agent MEMORY

## 모바일 화면 해상도 및 비율 대응 가이드라인

**🚀 핵심 레퍼런스: 클래시 로얄(Clash Royale) 스타일의 3단 세로형 분리 레이아웃**

현재 프로젝트(`머나먼약속`)는 모바일 세로형(Portrait) 전용 게임으로, 기기마다 다른 화면 비율(예: iPhone 15 Pro의 19.5:9 등)에 대응하기 위해 다음의 가이드라인과 클래시 로얄의 UI/UX 방식을 따릅니다.

1. **게임 뷰포트(Phaser) 고정**
   * Phaser 맵 크기는 기획 밸런스를 고려하여 `390x480`로 고정합니다. (가운데 정렬)
   * 전장 화면 자체는 늘어나거나 줄어들지 않고 절대적인 크기를 유지합니다.

2. **유연한 화면 전체 래퍼 유지**
   * 최상위 컨테이너는 `w-full min-h-[100dvh] max-w-[430px]` 등 가변적인 Flexbox로 구성하여 모든 기기 세로 해상도에 꽉 차도록 합니다. 절대 픽셀로 h-[844px]처럼 하드코딩하지 않습니다.

3. **기기별 빈 공간(레터박스)을 UI 컨테이너로 활용 (클래시 로얄 방식)**
   * Phaser 영역 위아래로 남는 빈 공간을 단순한 여백이 아닌 UI(상단 HUD, 하단 스킬/덱 스와이프 조작 등) 영역으로 활용합니다.
   * Flexbox의 남는 공간을 유연하게 차지하게 하되(`flex-1` 등), UI 컨텐츠를 화면 가장자리에 앵커링(Anchoring) 합니다.
   * **[상단]**: 게임 진행 시간, 획득 자원, 점령 상태도 등 정보성 HUD 배치
   * **[하단]**: 출격 대기 중인 덱(Deck), 스킬 버튼, 스와이프 조작 패드 등 조작계 배치

4. **안전영역 (Safe Area) 준수**
   * 아이폰의 다이내믹 아일랜드(노치)나 홈 인디케이터에 핵심 UI(버튼 등)가 가려지지 않도록 컨테이너 안쪽 여백에 `env(safe-area-inset-bottom)` / `env(safe-area-inset-top)` 등을 고려하여 설계합니다.
