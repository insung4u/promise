# ui-agent MEMORY

## 모바일 화면 해상도 및 비율 대응 가이드라인

**🚀 핵심 레퍼런스: 클래시 로얄(Clash Royale) 스타일의 3단 세로형 분리 레이아웃**

현재 프로젝트(`머나먼약속`)는 모바일 세로형(Portrait) 전용 게임으로, 기기마다 다른 화면 비율(예: iPhone 15 Pro의 19.5:9 등)에 대응하기 위해 다음의 가이드라인과 클래시 로얄의 UI/UX 방식을 따릅니다.

1. **게임 뷰포트(Phaser) 화면 100% 채움 (Full Screen)**
   * Phaser 맵은 절대 영역 구분이 아닌, 최상위 컨테이너 공간 전체를 `100% width, 100% height` (Phase.Scale.RESIZE) 로 채워서 배경처럼 랜더링 합니다.
   * 전투 보드가 화면 여백 없이 가득 채워지는 클래시 로얄의 깊은 몰입감을 재현합니다.

2. **유연한 화면 전체 래퍼 유지**
   * 최상위 컨테이너는 `relative w-full h-full max-w-[430px]` 로 구성하여 모든 기기 해상도에 꽉 차도록 합니다.

3. **기기별 빈 공간(레터박스) 대신 오버레이(Absolute) UI 활용**
   * 화면 위아래 남는 빈 공간을 막는 것이 아니라, **게임 맵 화면 위에 UI를 절대 위치(`absolute z-10`)로 공중에 띄우는 오버레이(Overlay) 방식**으로 덮습니다.
   * 빈 배경(`pointer-events-none`)을 통해 오버레이 뒤에 있는 맵 화면을 유저가 볼 수 있게 합니다.
   * **[상단 오버레이]**: 게임 진행 시간, 획득 자원, 점령 상태도 등 정보성 HUD를 상단에 고정
   * **[하단 오버레이]**: 출격 대기 중인 덱(Deck), 스킬 버튼, 스와이프 조작 패드 등 조작계를 하단에 고정

4. **안전영역 (Safe Area) 준수**
   * 아이폰의 다이내믹 아일랜드(노치)나 홈 인디케이터에 핵심 UI(버튼 등)가 가려지지 않도록 컨테이너 안쪽 여백에 `env(safe-area-inset-bottom)` / `env(safe-area-inset-top)` 등을 고려하여 설계합니다.
